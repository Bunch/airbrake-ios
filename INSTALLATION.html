Setting up your iOS app

  1. Download the notifier. The notifier is currently in private beta, so you should already have this link if you are in the beta. If you'd like to be notified when the iOS notifier launches, drop us a line in the support forum.
  2. Drag the hoptoadnotifier and kissxml folders to your project.
    • Make sure "Copy Items" and "Create Groups" are selected.
    • If you are already using kissxml, you don't need to include it again.
  3. Add SystemConfiguration.framework and libxml2.dylib to your project.
  4. Add the path /usr/include/libxml2 to Header Search Paths in your project's build settings.
    • Make sure you add it under "All Configurations".
  5. Import HTNotifier.h in your app delegate header file. 

    #import "HTNotifier.h"

  6. Add the following code to the very beginning of your application:didFinishLaunchingWithOptions:
    • Code executed before this line will not be monitored for exceptions and crashes

    [HTNotifier startNotifierWithAPIKey:@"89eca7626cf92e77a69843d3a9129faf"
                  environmentName:@"<# environment #>"];

Testing

  1. To test that the notifier is working inside your application, a simple test method is provided. This method creates a notice named [HTNotice selectorThatDoesNotExist]; with a sample backtrace and other appropriate fields. Add the following code to the very beginning of your application:didFinishLaunchingWithOptions: 

    2. [HTNotifier startNotifierWithAPIKey:@"89eca7626cf92e77a69843d3a9129faf"
                    environmentName:@"<# environment #>"];
[[HTNotifier startNotifier] writeTestNotice];

  2. That notice will be picked up by the notifier and reported just like a normal notice.
